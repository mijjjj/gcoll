# 本地验证指令

## 适用范围

本文件用于约束 AI 或开发者在本仓库完成代码修改后的本地验证方式。

## 验证原则

- 修改后端 Go 代码后，优先运行 `go test ./...`。
- 修改前端代码后，优先运行 `pnpm --dir frontend/web build`。
- 只要代码修改完成并且对应的后端测试、前端构建通过，即可作为本地验证通过。
- 除非用户明确要求，不需要启动 `gcoll-server`、Vite 开发服务、前端预览服务或桌面端运行时。
- 除非用户明确要求，不需要打开浏览器验证本地页面。

## 禁止默认执行

以下命令不得作为默认验证步骤自动执行：

- `go run ./cmd/gcoll-server`
- `pnpm --dir frontend/web dev`
- `pnpm --dir frontend/web preview`
- 任何会长期占用本地端口的前端或后端服务启动命令。

## 例外情况

只有在用户明确要求“启动服务”“打开页面”“用浏览器检查”“查看本地运行效果”或类似意图时，才可以启动对应前后端程序。验证完成后应主动关闭本次启动的进程。
