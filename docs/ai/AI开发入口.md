# AI 开发入口

本文件给后续 AI 执行 gcoll 开发任务使用。开始任务前先读 `docs/README.md`，再按任务类型读取对应规范。

## 默认上下文

- 项目语言和文档说明使用中文。
- 后端使用 Go + GoFrame。
- 桌面端使用 Wails 3。
- 前端使用 Vue 3 + TypeScript + Vite + Naive UI。
- 插件通信使用 gRPC + Protocol Buffers。
- 当前阶段是 MVP，内部契约按最新设计推进。

## AI 执行规则

1. 先识别任务是否影响公开契约。
2. 若影响 HTTP API、插件协议、SDK、事件格式或数据库迁移，先更新当前规范并说明兼容策略。
3. 若只影响内部实现，按最新设计直接推进，不保留旧内部模式。
4. 修改代码前先搜索现有模式，优先复用项目结构。
5. 新增文档和代码注释使用中文。
6. 实现后补充最小但有效的测试。
7. 前端实现后检查浅色、黑夜、空状态、错误状态和 `1280x800`。

## 常见边界提醒

- 控制面不处理高频采集数据。
- 数据面不依赖 HTTP 请求生命周期。
- 插件不得直接写宿主数据库。
- 南向插件不得直接转发最终目标系统。
- 北向插件不得主动采集设备数据。
- 设备配置页不维护点位表字段。
- 点位表不保存连接密码、Token 或密钥。
- 采集明细不落库。
