# ADR-0003 设备维度插件配置由宿主保存

## 状态

建议采纳。

## 背景

同一个插件可能服务多个设备，而每个设备的连接参数、采集周期、认证信息、点表文件和字段映射都可能不同。如果配置保存在插件进程或插件目录中，会难以审计、备份、加密、权限控制和版本管理。

## 决策

插件只声明配置结构，配置值由宿主保存：

- 插件通过 `configSchema` 声明配置项。
- 宿主根据配置结构生成默认表单。
- 每个设备拥有独立插件配置。
- 配置值保存到宿主数据库。
- 敏感值保存到密钥存储，数据库保存引用。
- 插件自定义配置页面只能通过宿主 API 读取和保存配置。

## 影响

正面影响：

- 配置治理集中。
- 支持权限、审计、版本和回滚。
- 同一插件可安全服务多个设备。
- 自定义页面无法绕过宿主写库。

负面影响：

- 宿主需要实现配置结构解析和表单渲染。
- 插件协议需要支持配置结构和配置校验。
- 自定义页面通信通道需要安全设计。

## 备选方案

- 插件自行保存配置：开发简单，但安全和治理能力弱。
- 所有配置都写到任务中：会导致设备、任务和插件配置耦合过重。

## 后续动作

- 实现 `plugin_config_schemas`。
- 实现 `plugin_device_configs`。
- 实现服务端配置校验和敏感字段处理。
- 实现配置变更审计。
