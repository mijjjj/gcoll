# ADR-0007 本地 HTTP API 使用 APIKey 授权

## 状态

已采纳。

## 背景

桌面端和本地运行时需要开放本地 HTTP API，便于本机脚本、调试工具和第三方程序访问。但本地 API 如果完全无鉴权，会带来误操作和本机越权访问风险。

## 决策

开启本地 HTTP API，并要求通过 APIKey 获取授权访问。

要求：

- 本地 HTTP API 默认开启。
- 所有非健康检查接口必须校验 APIKey。
- APIKey 应支持创建、吊销、过期时间和权限范围。
- APIKey 操作需要记录审计日志。

## 影响

正面影响：

- 便于本机集成和自动化调试。
- 安全边界比无鉴权更清晰。

负面影响：

- 需要实现 APIKey 生命周期管理。
- 用户需要保存和轮换 APIKey。

## 后续动作

- 设计 APIKey 数据表。
- 实现 APIKey 鉴权中间件。
- 在前端提供 APIKey 管理页面。
