# ADR-0008 用户权限简化为只读和可编辑

## 状态

已采纳。

## 背景

当前阶段不需要多租户、复杂 RBAC、组织结构和资源配额。过早引入复杂权限模型会增加前后端实现成本和测试成本。

## 决策

用户体系只支持两类账号：

- 只读账号：只能查看设备、任务、插件、配置、运行状态和日志。
- 可编辑账号：可以新增、修改、删除设备、任务和插件配置，并执行插件启停等管理操作。

## 影响

正面影响：

- 权限模型简单清晰。
- 实现成本低。
- 更适合 MVP 快速闭环。

负面影响：

- 无法表达复杂企业权限。
- 无法按设备、插件或任务做细粒度授权。

## 后续动作

- 用户表保留账号类型字段。
- API 中间件按只读和可编辑校验权限。
- 前端按账号类型隐藏或禁用编辑操作。
