# 产品范围

## 产品定位

gcoll 是同时支持服务器部署和桌面端部署的数据采集平台。它面向工业现场、企业集成、边缘网关、本地调试和轻量运维场景，帮助用户从设备、协议、文件、数据库、消息队列或业务系统采集数据，经本地或中心侧处理后转发到目标系统。

系统使用同一套核心运行时支撑两种形态：

- 服务器部署：长期运行的集中式服务，提供采集调度、本地插件托管、管理 API、Web 控制台和应用商城扩展接口。
- 桌面端部署：通过 Wails 3 打包为桌面应用，用于本机部署、边缘采集、离线运行、现场调试和轻量运维。

## 目标用户

- 运维工程师：配置采集任务、查看运行状态、处理告警和日志。
- 系统集成商：开发南向采集插件、北向转发插件和系统扩展插件。
- 企业管理员：管理用户权限、本地插件、设备、任务和应用商城源。
- 边缘部署工程师：在现场环境使用桌面端或轻量服务器完成本地采集和转发。

## MVP 核心目标

- 使用同一套核心运行时支持服务器端和桌面端。
- 建立插件化采集、处理、转发闭环。
- 当前阶段只支持本地插件模式，预留在线应用商城和企业私有源。
- 插件分为系统插件、南向插件、北向插件。
- 插件默认独立进程运行，通过 gRPC 与宿主通信。
- 插件配置由宿主按设备维度保存、校验、审计和加密敏感字段。
- 所有设备默认使用通用点位表。
- 采集数据不落库，只缓存最新点位数据。
- 数据按上报模式和规则过滤后再交给北向插件。
- 用户权限先简化为只读和可编辑。

## MVP 暂不做

- 完整在线应用商城、评论评分、作者主页和社区能力。
- 多租户计费。
- 低代码可视化数据管道编排。
- 原生云市场集成。
- 采集历史数据存储和查询。
- 外部队列中间件。
- 第一个稳定版本发布前，对内部插件 API 做长期兼容承诺。

## 核心用户流程

### 配置采集转发链路

1. 安装或启用一个南向插件。
2. 创建设备并选择插件。
3. 配置设备插件参数。
4. 配置通用点位表或插件自带点位页面。
5. 创建采集任务。
6. 配置规则过滤和北向转发目标。
7. 启动链路。
8. 查看任务健康状态、吞吐、延迟和错误日志。

### 本地插件安装

1. 用户导入插件包。
2. 宿主校验插件包签名、校验和、兼容性和权限声明。
3. 宿主注册插件元数据。
4. 用户启用插件。
5. 宿主启动插件进程并执行握手和健康检查。

### 桌面端本地采集

1. 用户启动桌面应用。
2. 内嵌运行时启动本地 API、SQLite 和插件宿主。
3. 用户配置本地插件、设备、点位和采集任务。
4. 运行时采集数据并更新最新点位缓存。
5. 网络恢复后由北向插件或同步模块处理补发。

## 关键指标

- 采集任务在线率。
- 采集成功率。
- 最新点位缓存更新次数。
- 点位启用数量。
- 规则过滤命中率。
- 北向插件接收成功率。
- 端到端数据延迟。
- 插件崩溃次数。
- 插件重启次数。
- 运行时 CPU 和内存使用率。
- 本地插件安装成功率。

## 第一个里程碑

第一个里程碑只验证完整闭环：

1. 启动 `gcoll-server`。
2. 本地安装一个南向采集插件。
3. 本地安装一个北向 HTTP 转发插件。
4. 创建设备、点位和采集任务。
5. 采集样例点位数据并标准化。
6. 通过规则过滤后转发给北向插件。
7. 在前端查看运行状态和插件日志。
