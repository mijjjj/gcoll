# 插件与点位协议

## 插件类型

| 类型 | 职责 | 示例 |
| --- | --- | --- |
| 系统插件 | 扩展平台内部能力 | 认证扩展、授权策略、告警通道、诊断扩展、调度策略 |
| 南向插件 | 从外部设备、协议、文件、数据库或 API 采集数据 | Modbus、OPC UA、MQTT 订阅、HTTP 轮询、文件监听 |
| 北向插件 | 将过滤后的数据转发到目标系统 | HTTP 转发、MQTT 发布、数据库写入、文件导出、云 IoT |

南向插件不得直接转发到最终目标系统。北向插件不得主动采集设备数据。系统插件按能力类型获得更高权限，安装时必须更严格校验。

## 插件包

插件包扩展名使用 `.gcpkg`，本质是固定结构的 zip 包。

```text
plugin.gcpkg
  plugin.yaml
  bin/
  assets/
  docs/
  checksums.txt
  signature.sig
```

`plugin.yaml` 中说明文本使用中文，机器字段保持英文。

```yaml
schemaVersion: 1
id: com.example.http-collector
name: HTTP Collector
type: southbound
version: 0.1.0
runtime: process
protocol: grpc
entry:
  windows-amd64: bin/plugin.exe
capabilities:
  - southbound.collector.http
permissions:
  - network.outbound
  - secret.read
compatibility:
  core: ">=0.1.0 <0.2.0"
  pluginProtocol: "1.x"
configSchema:
  type: object
  required:
    - url
  properties:
    url:
      type: string
      title: 接口地址
      ui:
        component: text
      rules:
        maxLength: 500
customConfigPage:
  enabled: false
signature:
  algorithm: ed25519
```

## 生命周期

```text
Discovered
  -> Verified
  -> Installed
  -> Enabled
  -> Starting
  -> Running
  -> Stopping
  -> Stopped
  -> Disabled
  -> Uninstalled
```

失败状态包括 `VerificationFailed`、`InstallFailed`、`StartFailed`、`RuntimeFailed`、`UpgradeFailed`、`RollbackFailed`。

升级流程必须先校验包，再安装候选版本，启动成功并通过健康检查后才能提升为活跃版本。失败时回滚到上一活跃版本。

## 插件协议

插件通信只支持 gRPC + Protocol Buffers。插件协议与管理 API 独立版本化。

通用上下文：

```proto
message RequestContext {
  string request_id = 1;
  string trace_id = 2;
  string device_id = 3;
  string task_id = 4;
  string plugin_id = 5;
  string plugin_version = 6;
  int64 deadline_unix_ms = 7;
}
```

通用服务：

```proto
service PluginRuntimeService {
  rpc Handshake(HandshakeRequest) returns (HandshakeResponse);
  rpc GetManifest(GetManifestRequest) returns (GetManifestResponse);
  rpc GetConfigSchema(GetConfigSchemaRequest) returns (GetConfigSchemaResponse);
  rpc ValidateConfig(ValidateConfigRequest) returns (ValidateConfigResponse);
  rpc Configure(ConfigureRequest) returns (ConfigureResponse);
  rpc Start(StartRequest) returns (StartResponse);
  rpc Stop(StopRequest) returns (StopResponse);
  rpc Health(HealthRequest) returns (HealthResponse);
  rpc SetDebugMode(SetDebugModeRequest) returns (SetDebugModeResponse);
}
```

南向服务：

```proto
service SouthboundService {
  rpc TestConnection(TestConnectionRequest) returns (TestConnectionResponse);
  rpc StartCollection(StartCollectionRequest) returns (StartCollectionResponse);
  rpc StopCollection(StopCollectionRequest) returns (StopCollectionResponse);
  rpc ReadOnce(ReadOnceRequest) returns (ReadOnceResponse);
}

service HostIngestService {
  rpc SubmitRecords(stream SubmitRecordsRequest) returns (SubmitRecordsResponse);
}
```

北向服务：

```proto
service NorthboundService {
  rpc TestTarget(TestTargetRequest) returns (TestTargetResponse);
  rpc Forward(ForwardRequest) returns (ForwardResponse);
}
```

## 采集记录

```proto
message CollectionRecord {
  string record_id = 1;
  string source_plugin_id = 2;
  string device_id = 3;
  string task_id = 4;
  string point_id = 5;
  int64 source_time_unix_ms = 6;
  int64 receive_time_unix_ms = 7;

  bytes raw_payload = 9;
  bytes normalized_payload = 10;
  string trace_id = 11;
  map<string, string> tags = 12;

```

`quality` 取值为 `good`、`uncertain`、`bad`。

## 设备维度配置

插件配置按设备独立保存。同一个插件用于多个设备时，每个设备拥有独立配置、独立版本和独立审计记录。

配置作用域：

- 插件默认配置：来自插件声明，不作为最终运行配置。
- 设备插件配置：MVP 的最终运行配置。
- 任务覆盖配置：后续可选能力。

配置字段至少支持：

- `number`
- `text`
- `textarea`
- `password`
- `select`
- `multiSelect`
- `switch`
- `file`
- `directory`

字段限制至少支持：

- `required`
- `minimum`
- `maximum`
- `minLength`
- `maxLength`
- `pattern`
- `enum`
- `accept`
- `maxFileSize`

宿主必须在服务端执行最终校验。前端校验只用于交互体验。

## 上报模式

设备插件配置必须支持上报模式：

- `immediate`：采集到数据后立即进入规则过滤。
- `change`：只有最新点位值变化后才进入规则过滤。

默认值为 `change`。变化判断基于最新点位缓存，不写历史采集明细。

## 自定义配置页面

插件可声明自定义配置页面，用于复杂配置体验。约束如下：

- 页面资源必须来自已安装并通过签名验证的插件包。
- 页面必须嵌入宿主隔离容器。
- 页面不得直接读写数据库和密钥文件。
- 页面只能通过宿主消息通道读取、校验、保存当前设备的当前插件配置。
- 页面加载、保存、测试连接等操作必须记录审计事件。
- 页面允许联网，联网安全性由用户负责。
- 黑夜模式、语言、权限状态由宿主消息通道同步。

## 通用点位表

所有设备默认使用通用点位表。点位表描述采集点，不描述连接参数。

推荐字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | string | 是 | 点位 ID，由宿主生成 |
| `device_id` | string | 是 | 设备 ID |
| `plugin_id` | string | 是 | 插件 ID |
| `name` | string | 是 | 点位名称 |
| `description` | string | 否 | 点位描述 |
| `address` | string | 是 | 点位地址，由插件解释 |
| `value_type` | string | 是 | 值类型 |
| `unit` | string | 否 | 单位 |
| `enabled` | bool | 是 | 是否启用 |
| `tags` | object | 否 | 标签 |
| `metadata` | object | 否 | 插件扩展元数据 |

内置值类型：

- `bool`
- `int`
- `float`
- `string`
- `bytes`
- `datetime`
- `json`

`metadata` 只能保存插件扩展信息，不得保存密码、Token 或密钥。

## 插件自带点位页面

只有通用点位表无法表达点位模型时，插件才可提供自带点位页面。页面只负责特殊点位配置，不替代设备配置页。

插件自带点位页面必须通过宿主点位 API 保存数据，不得直接写数据库。

## 内置 Modbus TCP 南向插件

首个本地南向插件为 `com.gcoll.modbus-tcp`，用于通过 Modbus TCP 采集和写入工业设备点位。

### 协议范围

- 使用 Modbus TCP 客户端模型连接设备，默认端口为 `502`。
- 支持读取 `coil`、`discrete_input`、`holding_register` 和 `input_register`。
- 支持写入 `coil` 和 `holding_register`；`discrete_input` 与 `input_register` 必须保持只读。
- 单次批量读取不得超过 Modbus 协议限制：线圈/离散输入最多 `2000` 位，寄存器最多 `125` 个。
- 单次批量写入不得超过协议限制：线圈最多 `1968` 位，寄存器最多 `123` 个。

### 配置边界

设备连接配置由宿主保存并下发，至少包含：

- `host`
- `port`
- `unitId`
- `timeoutMs`
- `pollIntervalMs`
- `reportMode`
- `maxCoilBatch`
- `maxRegisterBatch`
- `debugEnabled`

缺失 `host`、非法端口、非法 `unitId` 或非法批量长度必须报错，不得隐式选择其他设备、其他插件配置或旧配置。

### 点位 metadata

通用点位表仍保存平台通用字段。Modbus 扩展字段放入 `metadata`：

| 字段 | 说明 |
| --- | --- |
| `area` | `coil`、`discrete_input`、`holding_register`、`input_register` |
| `mode` | `read` 或 `write` |
| `quantity` | 点位占用的线圈或寄存器数量 |
| `valueType` | 插件侧解码使用的值类型，例如 `bool`、`uint16`、`float32` |
| `byteOrder` | `big` 或 `little` |
| `wordOrder` | `big` 或 `little` |
| `scale` | 数值缩放系数 |
| `offset` | 数值偏移 |

读取模式下，插件必须从宿主读取设备配置和点位表，并按 `area`、`address`、`id` 对启用读点位重新排序。插件应合并同一区域连续地址形成批量读取请求，并根据近期读取耗时、超时和错误动态调整批量长度；网络表现稳定时可以逐步增加，出现高延迟或错误时必须降低。

上报模式由设备配置决定：

- `change`：只有最新点位值变化后提交给宿主。
- `all`：每次读取成功都提交给宿主。

调试模式下，插件可以记录请求区域、起始地址、读取长度、耗时、原始响应摘要和错误信息。调试日志属于运行诊断数据，只允许保存有限窗口，不得保存敏感配置明文，也不得把采集明细作为历史数据长期落库。

## 安全和权限

插件权限建议分类：

- `network.outbound`
- `network.inbound`
- `filesystem.read`
- `filesystem.write`
- `device.serial`
- `device.usb`
- `secret.read`
- `config.read`
- `config.write`
- `runtime.metrics`
- `runtime.events`
- `system.auth`
- `system.storage`

宿主安装前展示权限，启动前校验权限。日志中不得输出敏感配置明文。

## 版本策略

- 补充可选字段：小版本升级。
- 新增可选服务方法：小版本升级。
- 删除字段、改变语义、改变必填字段：大版本升级。
- 宿主必须声明支持的插件协议版本范围。
