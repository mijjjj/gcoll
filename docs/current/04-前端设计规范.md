# 前端设计规范

## 设计定位

gcoll 是运维型、工程型数采平台。界面强调稳定、清晰、低干扰和高信息密度，默认风格是“简洁边缘桌面工作台”。

界面不要做成营销首页、展示大屏或装饰型后台。首屏直接进入工作台或最近使用的设备页。

## 技术基线

- Vue 3。
- TypeScript。
- Vite。
- Naive UI。
- Pinia。
- Vue Router。
- ECharts 或 uPlot。
- Monaco Editor。
- lucide 图标。

## 视觉原则

- 主色使用天蓝色。
- 支持浅色和黑夜模式。
- 表格、分栏、表单优先；卡片只用于指标摘要、独立对象或空状态。
- 页面说明使用短句，避免长段落。
- 状态使用状态点、短标签和紧凑文本，不用大面积色块。
- 默认紧凑密度，但保留鼠标点击区域。
- 不使用装饰性大图、渐变大横幅、重阴影、复杂插画和卡片墙。
- 服务端和桌面端共享同一套克制、扁平、工程化的界面语言。
- 桌面端是第一视觉参照，更强调本机状态、连接测试、点位表和调试日志。
- 不为视觉丰富增加不存在的业务模块。

## 布局规则

### 服务端 Web 控制台

- 左侧固定导航宽度 `232px`。
- 顶部工具栏高度 `56px`。
- 主内容内边距 `20px`。
- 内容区不限制最大宽度，适合大屏运维。
- 列表页优先使用表格。
- 页面标题、说明和主操作尽量放在同一行。
- 主导航包括工作台、设备管理、采集任务、点位缓存、规则过滤、北向转发、插件中心、日志诊断和系统设置。
- 底部导航包括 APIKey、用户权限和关于系统。

### 桌面端

- 基础视口按 `1280x800` 设计。
- 左侧导航宽度 `208px`，支持折叠到 `64px`。
- 顶部状态栏显示本地运行时、数据库、HTTP API、插件进程和网络状态。
- 内容区内边距 `16px`。
- 核心设备页采用“左侧设备列表 + 右侧设备详情 + 底部点位表”。
- 底部状态条只显示版本、APIKey 状态和最近错误等低频但重要的信息，不承载主要操作。
- 设置页必须能查看本地运行时、数据库、本地 HTTP API、APIKey、插件目录、主题和关于信息。

## 字体与层级

- 字体使用系统默认中文字体栈，Windows 优先 `Microsoft YaHei UI`。
- 页面标题使用 `20px / 28px`，字重 `600`。
- 分区标题使用 `16px / 24px`，字重 `600`。
- 表格正文使用 `13px / 20px`。
- 表单正文使用 `14px / 22px`。
- 辅助文本使用 `12px / 18px`。
- 字间距保持 `0`。
- 卡片和面板圆角不超过 `8px`。
- 按钮圆角 `6px`。
- 表格容器圆角 `8px`。
- 默认不使用厚重阴影；浅色模式使用边框表达层级，黑夜模式使用背景层级和边框表达层级。

## 组件规则

| 场景 | 推荐组件 | 要求 |
| --- | --- | --- |
| 导航 | `NMenu` | 扁平列表，选中项使用天蓝色左边线和浅底 |
| 表格 | `NDataTable` | 支持列设置、筛选、排序和批量选择 |
| 表单 | `NForm` | 标签左对齐，复杂配置按分组拆分 |
| 抽屉 | `NDrawer` | 用于设备、点位、插件、任务详情 |
| 弹窗 | `NModal` | 用于确认、导入和危险操作 |
| 标签 | `NTag` | 表示插件类型、状态、权限 |
| 分段控制 | `NSegmented` | 用于模式切换和上报模式 |
| 开关 | `NSwitch` | 用于启用状态和调试模式 |
| 图表 | ECharts/uPlot | 低装饰、高可读、网格线轻 |

图标按钮优先使用 lucide 图标，并提供悬停提示。每个页面最多突出一个主要操作。

## 页面模板

管理页面默认结构：

```text
页面标题区
筛选或工具条
主数据区域
详情抽屉或编辑弹窗
底部批量操作条
```

表格默认能力：

- 列排序。
- 列筛选。
- 关键词搜索。
- 批量选择。
- 列显示设置。
- 刷新。

抽屉默认结构：

```text
顶部：标题 / 状态 / 关闭
主体：标签页或分区表单
底部：保存 / 取消 / 危险操作
```

抽屉宽度：

- 服务端使用 `720px` 到 `960px`。
- 桌面端使用窗口宽度的 `56%`，最小 `640px`。

表单分组：

- 设备配置表单分为基本信息、插件选择、连接参数、认证信息、采集设置和调试设置。
- 点位表单分为基本信息、地址和值类型、单位和标签、插件扩展元数据。
- 设备配置表单不得混入点位字段。
- 桌面端表单默认只展开常用配置，大量插件配置项进入高级设置或折叠分组。

## 核心页面

### 工作台

展示运行设备、启用点位、运行任务、插件进程、转发速率、实时吞吐、最近告警和插件状态。不要做营销式大屏。

### 设备管理

服务端使用设备表格 + 右侧抽屉。桌面端使用左侧设备列表 + 右侧详情。

- 左侧设备列表顶部主加号支持“添加设备”和“添加分组”。
- 左侧分组头部支持直接“添加设备”和“移入现有设备”。
- 左侧空白区域右键菜单至少支持“添加分组”。
- 左侧分组和设备右键菜单至少支持“添加设备”；设备右键菜单还应支持“删除设备”和“移动到其他分组”。
- 点击左侧设备后默认进入“设备配置页”。
- 设备详情区必须提供“设备配置页”和“设备点位表”两个标签页切换。
- 设备配置页不编辑点位表，点位表放独立标签页或分区。

### 采集任务

展示任务名称、设备、南向插件、点位数、上报模式、状态、采集速率、规则命中率和最后采集时间。操作包括启动、停止、重启、查看日志和编辑。

### 点位缓存

只展示最新点位缓存，不提供历史查询。当前值列使用等宽数字，变化状态使用轻量高亮。

### 规则过滤

使用规则列表、详情抽屉、条件编辑器和测试样本区。后续可加入 Monaco Editor 高级模式。

### 北向转发

展示转发目标、路由规则、插件调用状态和最近错误。不展示目标系统投递成功率，除非北向插件主动上报。

### 插件中心

当前阶段突出本地插件导入、启停、升级、回滚和调试。在线应用商城只作为弱入口。

### 日志诊断

支持来源、级别、插件、设备、任务和时间范围筛选。日志流支持暂停、继续、导出和清空视图。

## 状态表达

| 状态 | 表达 |
| --- | --- |
| 运行中 | 绿色状态点 + `运行中` |
| 已停止 | 灰色状态点 + `已停止` |
| 离线 | 红色状态点 + `离线` |
| 告警 | 黄色状态点 + 告警数量 |
| 调试中 | 天蓝色 `调试中` 标签 |
| 变化上报 | 蓝色弱标签 |
| 立即上报 | 橙色弱标签 |

状态颜色不得作为唯一信息来源，必须配合文字。

## 交互与权限

- 操作成功使用右上角轻提示。
- 操作失败展示错误提示和可展开详情。
- 长任务显示进度条或步骤。
- 插件启动显示启动中状态和超时提示。
- 日志流默认自动滚动，可暂停。
- 实时图表默认 5 秒刷新。
- 连接测试结果在设备页就地显示，不跳转到日志页。
- 只读账号隐藏新增、编辑、删除、启停、导入和回滚按钮，表格操作列只保留查看，表单进入只读态。
- 可编辑账号可访问配置和管理操作，危险操作仍需要确认。
- APIKey 页面必须表达正常、即将过期、已过期、已吊销和最近异常访问状态。

## 插件自定义页面

- 外层保留宿主标题、说明、保存状态、权限提示和审计提示。
- 插件页面置于独立容器中。
- 容器顶部显示插件名称、版本、权限和联网提示。
- 插件自定义配置页如果已经在页面内部提供保存和测试按钮，宿主外层不得再重复渲染同名主操作按钮。
- 系统默认简单配置页由宿主根据 `configSchema` 渲染字段、默认值、保存按钮和测试按钮。
- 黑夜模式通过宿主消息通道同步给插件页面。
- 插件页面加载失败、版本不兼容、请求越权和保存失败由宿主统一展示。

## 可用性检查

每个页面实现后必须检查：

- 浅色和黑夜模式文字对比度。
- `1280px` 宽度下表格列不严重挤压。
- 按钮文字不溢出。
- 抽屉底部操作始终可见。
- 日志和表格长文本有省略和详情查看。
- 插件自定义页面能同步主题和权限状态。
