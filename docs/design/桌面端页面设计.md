# gcoll 桌面端页面设计

## 1. 设计目标

桌面端用于本机部署、边缘采集、现场调试和轻量运维。相比服务端，桌面端更强调本机状态、快速配置、调试反馈和离线可用。

桌面端采用“简洁边缘桌面工作台”风格。界面优先服务现场配置和调试，不追求装饰性视觉。核心页面应在一个窗口内完成主要任务，减少页面跳转。

基础视口：

- 推荐最小尺寸：`1280x800`。
- 默认布局：左侧窄导航 + 顶部本机状态栏 + 内容区。
- 支持黑夜模式。

## 2. 桌面端框架

```text
窗口标题栏：gcoll Desktop / 运行状态 / 主题切换 / 设置
左侧导航：工作台、设备、任务、点位缓存、插件、日志、设置
顶部状态栏：本地运行时、数据库、HTTP API、插件进程、网络
内容区：当前页面
底部状态条：版本、APIKey 状态、最近错误
```

框架要求：

- 左侧导航默认展开，支持折叠。
- 顶部状态栏始终可见，不随页面滚动。
- 内容区避免多层卡片，优先使用分栏、表格和轻量边框。
- 底部状态条只显示低频但重要的信息，不承载主要操作。

## 3. 工作台

### 3.1 页面目标

让现场工程师打开应用后立即看到本机运行状态。

### 3.2 页面布局

- 本机运行时状态条。
- 快速指标：设备数、启用点位、运行任务、插件进程、最近错误。
- 最近 15 分钟采集速率图。
- 本地插件状态表。
- 快捷操作区。

快捷操作：

- 新建设备。
- 导入插件。
- 打开本地 HTTP API 设置。
- 查看调试日志。

## 4. 设备页

桌面端设备页采用左右分栏：

```text
左栏：设备列表和分组
右栏上部：设备详情和连接测试
右栏下部：采集点位列表
```

左栏宽度 `320px`，支持搜索和状态筛选。右栏包含：

- 基本信息。
- 设备配置。
- 点位表。
- 运行状态。
- 调试日志。

设备页是桌面端的核心页面，应优先打磨。用户应能在该页面完成设备选择、连接配置、连接测试、点位维护、启停任务和查看实时值。

### 4.1 设备配置

以表单为主：

- 插件。
- 连接地址。
- 认证信息。
- 采集周期。
- 上报模式。
- 调试模式。
- 测试连接。

桌面端建议在表单右侧显示“连接测试结果”小面板，便于现场调试。

表单设计要求：

- 默认显示常用字段，高级字段放入“高级设置”标签页。
- 主操作只保留“保存”和“测试连接”。
- 危险操作进入更多菜单或确认弹窗。
- 启用状态、调试模式、上报模式必须靠近采集设置。

### 4.2 点位表

点位表使用紧凑表格，默认列：

- 名称。
- 地址。
- 值类型。
- 当前值。
- 质量。
- 更新时间。
- 启用。

桌面端提供更醒目的导入入口：

- 导入 CSV。
- 导入 Excel。
- 从插件扫描。

如果插件提供自带点位页面，直接在点位表区域嵌入，并保留返回通用点位表的入口。

## 5. 采集任务页

桌面端采集任务页强调启动、停止和错误定位。

布局：

- 顶部任务状态概览。
- 任务列表。
- 任务详情抽屉。

任务详情显示：

- 绑定设备。
- 点位数量。
- 上报模式。
- 采集速率。
- 最新错误。
- 最近日志。

操作按钮：

- 启动。
- 停止。
- 重启。
- 打开调试日志。

## 6. 插件页

桌面端插件页重点支持本地插件导入和调试。

页面结构：

- 顶部导入插件包区域。
- 插件列表。
- 插件详情。
- 调试日志。

插件卡片采用紧凑列表，不做大卡片墙。

插件页不做应用商城式视觉，当前阶段只突出本地插件管理；在线应用商城入口保留为弱入口或占位页。

列表字段：

- 插件名称。
- 类型。
- 版本。
- 状态。
- 调试模式。
- 操作。

操作：

- 启用。
- 禁用。
- 启动。
- 停止。
- 回滚。
- 打开目录。

## 7. 点位缓存页

桌面端点位缓存页用于现场实时观察。

设计：

- 顶部设备选择器。
- 点位搜索。
- 当前值表格。
- 变化高亮。

变化高亮规则：

- 值变化后整行使用天蓝色浅底闪烁一次。
- 黑夜模式下使用深蓝描边，不使用大面积亮色。
- 高亮持续不超过 `1.2s`。

## 8. 日志页

桌面端日志页默认展示本机日志。

筛选：

- 运行时。
- 插件。
- 设备。
- 任务。
- 调试日志。
- 错误日志。

操作：

- 暂停。
- 清屏。
- 导出。
- 打开日志目录。

## 9. 设置页

设置页分组：

- 本地运行时。
- 数据库。
- 本地 HTTP API。
- APIKey。
- 插件目录。
- 主题。
- 关于。

本地 HTTP API 设置：

- 开启状态。
- 监听地址。
- 端口。
- APIKey 管理。
- 最近访问记录。

主题设置：

- 跟随系统。
- 浅色。
- 黑夜。

## 10. 桌面端差异化规则

- 所有危险操作优先使用确认弹窗，不使用跳转确认页。
- 配置编辑优先就地完成。
- 日志和连接测试要比服务端更显眼。
- 离线状态要明确展示。
- 插件目录、本地数据库、本地 API 状态必须在设置页可见。
