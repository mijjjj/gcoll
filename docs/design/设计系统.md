# gcoll 页面设计系统

## 1. 设计定位

gcoll 是运维型、工程型数采平台，界面需要强调稳定、清晰、低干扰和高信息密度。视觉风格采用扁平现代化设计，主色为天蓝色，支持浅色模式和黑夜模式。

当前确认的主风格为“简洁边缘桌面工作台”。桌面端是第一视觉参照，服务端 Web 控制台沿用同一套克制、扁平、工程化的界面语言。整体应更像现场工程师每天使用的配置和调试工具，而不是营销后台或展示大屏。

设计关键词：

- 扁平。
- 现代。
- 工程感。
- 简洁。
- 高可读。
- 高密度但不拥挤。
- 状态清晰。
- 操作路径短。

不采用营销式大首页，不使用装饰性大图、渐变大横幅、夸张卡片、重阴影或复杂装饰。首屏直接进入工作台或最近使用的设备页。

## 2. 简洁原则

- 页面只保留当前任务需要的信息，不为填充视觉而增加模块。
- 导航层级最多两级，常用入口固定可见，低频能力进入设置或详情抽屉。
- 表格、分栏和表单优先，卡片只用于指标摘要、独立对象或空状态。
- 一屏内避免同时出现过多按钮，次要操作进入更多菜单。
- 状态信息使用状态点、短标签和紧凑文本，不使用大面积色块。
- 页面说明文字保持短句，避免长段落说明。
- 默认密度偏紧凑，但保留足够点击区域，适合鼠标操作。

## 3. 技术基线

推荐前端技术：

- Vue 3。
- TypeScript。
- Vite。
- Naive UI。
- Pinia。
- Vue Router。
- ECharts 或 uPlot。
- Monaco Editor。
- lucide 图标。

## 5. 布局规则

### 5.1 服务端 Web 控制台

- 左侧固定导航，宽度 `232px`。
- 顶部工具栏高度 `56px`。
- 内容区最大宽度不限制，适合大屏运维。
- 主内容内边距 `20px`。
- 列表页优先使用表格，不使用大面积卡片墙。
- 页面标题区高度紧凑，标题、说明、主操作同一行。
- 服务端页面也应避免“概览卡片堆叠”，工作台以指标条、趋势图和异常列表为主。

### 5.2 桌面端

- 左侧导航宽度 `208px`，支持折叠到 `64px`。
- 顶部状态栏显示本地运行时、APIKey、数据库、插件进程和网络状态。
- 内容区内边距 `16px`。
- 更多使用抽屉、分栏和就地编辑，减少窗口跳转。
- 适配 `1280x800` 作为桌面端基础视口。
- 设备页采用“左侧设备列表 + 右侧设备详情 + 底部点位表”的主结构，作为桌面端首要体验模板。

## 6. 组件规则

| 组件 | 推荐 Naive UI 组件 | 设计要求 |
| --- | --- | --- |
| 导航 | `NMenu` | 扁平列表，选中项使用天蓝色左边线和浅底 |
| 表格 | `NDataTable` | 默认紧凑密度，支持列设置、筛选、排序 |
| 表单 | `NForm` | 标签左对齐，复杂配置用分组标题 |
| 抽屉 | `NDrawer` | 用于设备编辑、点位编辑、插件详情 |
| 弹窗 | `NModal` | 用于确认、导入、危险操作 |
| 标签 | `NTag` | 表示插件类型、状态、权限 |
| 状态点 | 自定义 + `NBadge` | 运行、离线、警告、调试中 |
| 分段控制 | `NSegmented` | 模式切换、浅色/黑夜、立即/变化上报 |
| 开关 | `NSwitch` | 启用状态、调试模式 |
| 图表 | ECharts/uPlot | 低装饰、高可读，网格线轻 |

组件使用原则：

- 图标按钮优先使用 lucide 图标，并提供悬停提示。
- 主要操作按钮每个页面最多突出一个。
- 表单字段按业务分组，复杂配置通过标签页或折叠分组拆开。
- 批量操作条只在选择数据后出现。
- 插件自定义页面外层保留宿主的统一工具栏和保存状态。

## 7. 字体与尺寸

- 字体：系统默认中文字体栈，Windows 优先 `Microsoft YaHei UI`。
- 页面标题：`20px / 28px`，字重 `600`。
- 分区标题：`16px / 24px`，字重 `600`。
- 表格正文：`13px / 20px`。
- 表单正文：`14px / 22px`。
- 辅助文本：`12px / 18px`。
- 字间距保持 `0`。

## 8. 圆角与阴影

- 卡片和面板圆角不超过 `8px`。
- 按钮圆角 `6px`。
- 表格容器圆角 `8px`。
- 默认不使用厚重阴影。
- 浅色模式使用边框表达层级。
- 黑夜模式使用背景层级和边框表达层级。

## 9. 状态表达

| 状态 | 颜色 | 表达 |
| --- | --- | --- |
| 运行中 | 绿色 | 状态点 + `运行中` |
| 已停止 | 灰色 | 状态点 + `已停止` |
| 离线 | 红色 | 状态点 + `离线` |
| 告警 | 黄色 | 状态点 + 告警数量 |
| 调试中 | 天蓝色 | `调试中` 标签 |
| 变化上报 | 蓝色弱标签 | `变化上报` |
| 立即上报 | 橙色弱标签 | `立即上报` |

## 10. 黑夜模式切换

黑夜模式入口：

- 服务端：顶部右侧主题切换按钮。
- 桌面端：标题栏右侧主题切换按钮，并同步到本地设置。

切换规则：

- 切换后立即生效。
- 用户偏好保存到本地。
- 图表、表格、代码编辑器、日志查看器必须同步主题。
- 插件自定义页面通过宿主消息通道接收当前主题。
