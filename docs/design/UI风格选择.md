# gcoll UI 风格选择

## 1. 已确认方向

当前确认采用第三版方向：**简洁边缘桌面工作台**。

该方向以 Wails 桌面端为第一视觉参照，同时反向约束服务端 Web 控制台。服务端和桌面端共享同一套设计系统，但桌面端更强调本机运行时、设备配置、连接测试、点位表和调试日志。

## 2. 风格关键词

- 简洁。
- 扁平。
- 天蓝色主基调。
- 工程工具感。
- 信息清晰。
- 表格优先。
- 状态明确。
- 少装饰。
- 少跳转。
- 支持黑夜模式。

## 3. 视觉取舍

### 3.1 保留

- 左侧稳定导航。
- 顶部本机状态条。
- 设备列表和设备详情左右分栏。
- 设备详情中的标签页。
- 下方紧凑点位表。
- 连接测试和调试日志面板。
- 浅色模式下的天蓝色选中态。
- 黑夜模式下的深蓝背景和天蓝高亮。

### 3.2 减少

- 大面积卡片。
- 厚重阴影。
- 装饰性背景。
- 复杂渐变。
- 过多入口按钮。
- 页面内长篇说明。
- 与当前任务无关的图表。

### 3.3 禁止

- 营销式首屏。
- 卡片套卡片。
- 为了视觉丰富而增加不存在的业务模块。
- 使用大插画表达空状态。
- 用颜色作为唯一状态信息。

## 4. 默认首页

桌面端默认首页建议为“设备管理”或“工作台”，二者都必须让用户在首屏看到：

- 本地运行时状态。
- SQLite 数据库状态。
- 本地 HTTP API 状态。
- 插件进程状态。
- 网络状态。
- 最近设备或最近任务。

如果用户最近正在配置设备，则优先回到该设备详情页。

## 5. 页面结构模板

### 5.1 桌面端设备页

```text
左侧导航
顶部本机状态条
主区域：
  左侧设备列表
  右侧设备详情
  底部点位表
底部状态条
```

### 5.2 服务端管理页

```text
左侧导航
顶部工具栏
主区域：
  筛选工具条
  数据表格
  右侧详情抽屉
```

服务端可以比桌面端显示更多批量管理能力，但视觉上仍保持同样的简洁密度。

## 6. 后续生成页面时的要求

- 所有原型、截图和代码实现都以该文档为视觉基准。
- 新增页面必须先说明它对应的主任务，避免堆功能。
- 实现 Vue 组件时优先使用 Naive UI 的表格、表单、抽屉、标签、开关和分段控制。
- 页面状态必须同时覆盖浅色和黑夜模式。
- 图标统一使用 lucide 图标。
- 不为服务端和桌面端设计两套完全不同的风格。
