# gcoll 页面交互与状态规范

## 1. 通用页面结构

每个管理页面默认结构：

```text
页面标题区
筛选/工具条
主数据区域
详情抽屉或编辑弹窗
底部批量操作条
```

简洁风格优先级：

1. 当前任务路径清楚。
2. 常用操作一步可达。
3. 状态反馈明确。
4. 次要信息可展开查看。
5. 视觉装饰尽量减少。

标题区包含：

- 页面标题。
- 简短说明。
- 主要操作按钮。
- 刷新按钮。

## 2. 表格规范

表格默认能力：

- 列排序。
- 列筛选。
- 关键词搜索。
- 批量选择。
- 列显示设置。
- 刷新。

表格密度：

- 服务端默认 `medium`。
- 桌面端默认 `small`。

空状态：

- 说明当前没有数据。
- 提供主要下一步操作。
- 不使用大插画。
- 不使用营销式文案。

## 3. 抽屉规范

抽屉用于：

- 设备详情。
- 点位编辑。
- 插件详情。
- 任务详情。
- 转发目标详情。

抽屉结构：

```text
顶部：标题 / 状态 / 关闭
主体：标签页或分区表单
底部：保存 / 取消 / 危险操作
```

宽度：

- 服务端：`720px` 到 `960px`。
- 桌面端：窗口宽度的 `56%`，最小 `640px`。

## 4. 表单规范

设备配置表单分组：

- 基本信息。
- 插件选择。
- 连接参数。
- 认证信息。
- 采集设置。
- 调试设置。

点位表单分组：

- 基本信息。
- 地址和值类型。
- 单位和标签。
- 插件扩展元数据。

设备配置表单不得混入点位字段。

桌面端表单默认只展开常用配置。插件提供的大量配置项应按分组或高级设置折叠展示，避免首屏压迫。

## 5. 插件自定义页面嵌入

嵌入区域要求：

- 外层保留宿主标题、说明、保存状态和审计提示。
- 插件页面置于独立容器中。
- 容器顶部显示插件名称、版本、权限和联网提示。
- 黑夜模式通过宿主消息通道同步给插件页面。

异常状态：

- 插件页面加载失败。
- 插件页面版本不兼容。
- 插件页面请求越权。
- 插件页面保存失败。

这些状态必须由宿主统一展示。

## 6. 状态和反馈

操作反馈：

- 成功：右上角轻提示。
- 失败：错误提示 + 可展开详情。
- 长任务：显示进度条或步骤。
- 插件启动：显示启动中状态和超时提示。

高频状态：

- 最新点位变化使用轻量高亮。
- 日志流默认自动滚动，可暂停。
- 实时图表默认 5 秒刷新。
- 连接测试结果在设备页就地显示，不跳转到日志页。

## 7. 黑夜模式细节

黑夜模式下：

- 页面背景使用深蓝黑，不使用纯黑。
- 表格边框降低亮度。
- 图表网格线透明度降低。
- 状态颜色保持可读，但降低饱和度。
- 代码编辑器和日志区域使用深色主题。

## 8. 权限状态

只读账号：

- 隐藏新增、编辑、删除、启停、导入、回滚按钮。
- 表格操作列只保留查看。
- 表单进入只读态。

可编辑账号：

- 可访问全部配置和管理操作。
- 危险操作仍需要确认。

## 9. APIKey 状态

APIKey 页面状态：

- 正常。
- 即将过期。
- 已过期。
- 已吊销。
- 最近异常访问。

本地 HTTP API 未授权访问时，页面展示明确说明和创建 APIKey 入口。

## 10. 视觉可用性检查

每个页面实现后需要检查：

- 浅色和黑夜模式文字对比度。
- 表格列在 `1280px` 宽度下不严重挤压。
- 按钮文字不换行溢出。
- 抽屉底部操作始终可见。
- 日志和表格长文本有省略和详情查看。
- 状态颜色不作为唯一信息来源，必须配合文字。
