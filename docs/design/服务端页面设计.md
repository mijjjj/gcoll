# gcoll 服务端页面设计

## 1. 信息架构

服务端 Web 控制台用于集中管理设备、点位、任务、插件和运行状态。布局采用左侧导航 + 顶部工具栏 + 内容区。

服务端沿用“简洁边缘桌面工作台”的视觉语言，但信息密度可以更高，批量管理能力可以更完整。页面不要做成展示型大屏，也不要使用营销式首页。

主导航：

- 工作台。
- 设备管理。
- 采集任务。
- 点位缓存。
- 规则过滤。
- 北向转发。
- 插件中心。
- 日志诊断。
- 系统设置。

底部导航：

- APIKey。
- 用户权限。
- 关于系统。

## 2. 工作台

### 2.1 页面目标

让运维人员快速判断系统是否健康，以及采集、规则过滤、北向转发是否正常。

### 2.2 页面布局

```text
标题区：工作台 / 时间范围 / 刷新 / 主题切换
状态条：运行设备、启用点位、运行任务、插件进程、转发到北向插件速率
主区域左：实时吞吐折线图
主区域右：系统健康摘要
下方左：最近告警
下方右：插件状态列表
```

### 2.3 关键组件

- 指标条：紧凑横向指标，不超过 `8px` 圆角。
- 实时图表：显示采集速率、规则命中、转发到北向插件速率。
- 健康摘要：数据库、插件宿主、本地插件目录、gRPC 协议状态。
- 告警列表：按时间倒序。

### 2.4 状态

- 正常：天蓝色和绿色为主。
- 有告警：对应指标卡显示黄色角标。
- 插件异常：插件状态列表置顶异常插件。
- 空状态：显示“暂无运行任务”，提供“创建采集任务”按钮。

## 3. 设备管理

### 3.1 页面目标

管理设备档案、设备插件配置、点位表和调试模式。

服务端设备页也应尽量靠近桌面端设备页结构，让用户在表格和详情抽屉之间完成配置，不进入多级页面。

### 3.2 页面布局

```text
标题区：设备管理 / 新建设备 / 导入
筛选条：插件类型、运行状态、上报模式、关键词
设备表格：名称、插件、点位数、上报模式、状态、最后更新、操作
右侧抽屉：设备详情
```

### 3.3 设备详情抽屉

抽屉宽度建议 `720px`，分为标签页：

- 基本信息。
- 设备配置。
- 点位表。
- 运行状态。
- 调试日志。

### 3.4 设备配置标签页

内容：

- 插件选择。
- 连接参数。
- 认证参数。
- 采集周期。
- 上报模式：立即上报 / 变化上报，默认变化上报。
- 调试模式开关。
- 测试连接按钮。

设备配置页不编辑点位表。

### 3.5 点位表标签页

默认使用通用点位表。

表格列：

- 点位名称。
- 描述。
- 地址。
- 值类型。
- 单位。
- 启用状态。
- 标签。
- 最近值。
- 更新时间。
- 操作。

操作：

- 新增点位。
- 批量导入。
- 批量导出。
- 批量启用/停用。
- 校验点位。

当插件声明自带点位页面时：

- 点位表顶部显示“该插件使用自带点位页面”提示。
- 页面嵌入到点位表标签页。
- 宿主仍保留导入、导出、校验入口。

## 4. 采集任务

### 4.1 页面目标

管理采集任务生命周期，并查看任务吞吐、延迟和错误。

### 4.2 表格列

- 任务名称。
- 设备。
- 南向插件。
- 点位数。
- 上报模式。
- 状态。
- 采集速率。
- 规则命中率。
- 最后采集时间。
- 操作。

### 4.3 操作

- 启动。
- 停止。
- 重启。
- 查看日志。
- 查看点位缓存。
- 编辑任务。

危险操作使用二次确认。

## 5. 点位缓存

### 5.1 页面目标

查看运行时最新点位缓存，不提供历史数据查询。

### 5.2 页面布局

```text
筛选条：设备、插件、任务、值类型、变化状态、关键词
缓存表格：点位、设备、当前值、质量、值类型、更新时间、变化状态
右侧详情：点位元数据、最近一次采集上下文
```

### 5.3 设计规则

- 当前值列使用等宽数字。
- 变化状态使用小标签：已变化 / 未变化。
- 质量状态使用颜色点：good、uncertain、bad。
- 页面说明明确“不保存历史采集数据”。

## 6. 规则过滤

### 6.1 页面目标

配置采集数据进入北向插件前的过滤规则。

### 6.2 页面布局

- 规则列表。
- 规则详情抽屉。
- 条件编辑器。
- 测试样本区。

### 6.3 规则列表列

- 规则名称。
- 适用设备。
- 适用点位。
- 条件摘要。
- 命中次数。
- 状态。
- 更新时间。
- 操作。

### 6.4 条件编辑器

使用分组式规则构建器：

- 点位字段。
- 操作符。
- 比较值。
- 组合关系：全部满足 / 任一满足。

后续可增加 Monaco Editor 高级模式。

## 7. 北向转发

### 7.1 页面目标

管理北向插件目标和路由关系。框架只负责将规则过滤后的消息转发给北向插件。

### 7.2 页面结构

- 转发目标列表。
- 路由规则列表。
- 插件调用状态。
- 最近错误。

### 7.3 转发目标表格

- 名称。
- 北向插件。
- 目标类型。
- 状态。
- 最近调用时间。
- 插件接收成功率。
- 操作。

不展示目标系统投递成功率，除非北向插件主动上报。

## 8. 插件中心

### 8.1 页面目标

管理本地插件安装、启停、升级、回滚和调试。

当前阶段只支持本地插件模式。在线应用商城作为预留入口，不在首屏强化展示。

### 8.2 页面布局

```text
标题区：插件中心 / 导入插件包
筛选条：插件类型、状态、关键词
插件表格：名称、类型、版本、协议、权限、状态、调试、操作
详情抽屉：概览、配置结构、权限、日志、版本
```

插件类型标签：

- 系统插件。
- 南向插件。
- 北向插件。

### 8.3 插件详情

标签页：

- 概览。
- 权限。
- 配置结构。
- 版本历史。
- 调试日志。

调试模式开关放在详情页顶部，开启后显示日志限流提示。

## 9. 日志诊断

### 9.1 页面目标

集中查看运行时日志、插件日志和调试日志。

### 9.2 页面布局

- 左侧筛选：级别、来源、插件、设备、任务、时间范围。
- 右侧日志流。
- 顶部操作：暂停、继续、导出、清空视图。

黑夜模式下日志区域使用深色背景和高对比文本。

## 10. 系统设置

页面分组：

- 基础设置。
- 存储设置。
- 本地插件目录。
- APIKey。
- 用户权限。
- 主题设置。

用户权限只包含：

- 只读。
- 可编辑。

APIKey 页面需要支持：

- 创建。
- 查看最后使用时间。
- 吊销。
- 权限范围。
