# No Default Fallback

## 规则

不要为缺失、禁用、无权限或跨范围配置设计隐式兜底。

除非用户明确要求默认配置或兜底选项：

- 必需配置缺失时返回错误。
- 引用资源不存在时返回错误。
- 引用资源不属于当前设备、用户或部署范围时返回错误。
- 外部端点、凭证、插件、配置结构、点位表、密钥或功能配置缺失时返回错误。
- 不要静默回退到 `system`、硬编码 ID、第一条记录、环境默认值或旧配置行。

## 允许的默认值

只允许不会选择另一个业务资源、凭证、设备、插件或外部服务的本地值归一化，例如：

- 空分页大小归一化为项目分页默认值。
- 空超时时间归一化为安全请求超时。
- 空枚举值归一化，但前提是 API 契约已经定义该行为。

## 必查项

修改配置或资源查找代码时：

1. 搜索 `system`、`default`、`fallback`、`GetDefault`、`WhereOr` 和硬编码 ID。
2. 移除隐式兜底路径，除非本次用户需求明确要求保留。
3. 保持设备配置、点位表、插件配置结构和密钥引用分离。
4. 对缺失或无效配置返回清晰业务错误。
5. 如果主动拒绝或移除了兜底行为，在最终说明中写明。
