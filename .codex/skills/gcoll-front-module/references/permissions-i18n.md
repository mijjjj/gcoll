# 权限与 i18n 模式

## i18n

- 所有用户可见文本进入现有 i18n 结构。
- 中英文 key 层级保持一致。
- 通用动作、状态和占位符优先复用 `common` 类 key。
- 动态文本使用 `{name}` 这类变量形式。
- 不把 JSON、SQL、XML 或大量花括号示例直接放入 locale 字符串。

## 路由和菜单

- 路由 path 与后端菜单或权限定义保持一致。
- `meta.title` 使用 i18n key。
- 只读权限下隐藏或禁用编辑、删除、保存等操作，并保留可理解提示。

## 插件自定义页面

- 外层保留宿主工具栏。
- 显示保存状态、权限提示和审计提示。
- 插件页面不得绕过宿主主题、权限和配置校验。

## 检查点

- 浅色模式。
- 黑夜模式。
- 空状态。
- 错误状态。
- 只读权限状态。
- `1280x800` 视口。
