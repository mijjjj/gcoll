# gcoll 前端设计执行指南

本指南用于实现 `frontend/web` 页面和后续 Wails 桌面端页面。当前视觉方向是“简洁边缘桌面工作台”：扁平、现代、工程感、高密度但不拥挤，主色为天蓝色，支持浅色和黑夜模式。参考图保存在 `../assets/desktop-workbench-reference.png`。

## 视觉取舍

- 保留左侧稳定导航、顶部状态条、设备列表和设备详情左右分栏、详情标签页、下方紧凑点位表、连接测试和调试日志面板。
- 减少大面积卡片、厚重阴影、装饰背景、复杂渐变、过多入口按钮、长篇说明和与当前任务无关的图表。
- 禁止营销式首屏、卡片套卡片、为了视觉丰富增加不存在的业务模块、大插画空状态、仅用颜色表达状态。

## 字体与尺寸

- 字体使用系统默认中文字体栈，Windows 优先 `Microsoft YaHei UI`。
- 页面标题使用 `20px / 28px`，字重 `600`。
- 分区标题使用 `16px / 24px`，字重 `600`。
- 表格正文使用 `13px / 20px`。
- 表单正文使用 `14px / 22px`。
- 辅助文本使用 `12px / 18px`。
- 字间距保持 `0`。
- 卡片和面板圆角不超过 `8px`，按钮圆角 `6px`，表格容器圆角 `8px`。
- 默认不使用厚重阴影，浅色模式用边框表达层级，黑夜模式用背景层级和边框表达层级。

## 页面结构

管理页面默认结构：

```text
页面标题区
筛选或工具条
主数据区域
详情抽屉或编辑弹窗
底部批量操作条
```

表格默认支持列排序、列筛选、关键词搜索、批量选择、列显示设置和刷新。服务端表格默认 `medium` 密度，桌面端默认 `small` 密度。空状态说明当前没有数据并提供主要下一步操作，不使用大插画和营销式文案。

抽屉用于设备详情、点位编辑、插件详情、任务详情和转发目标详情。抽屉结构为顶部标题和状态、主体标签页或分区表单、底部保存取消和危险操作。服务端抽屉宽度建议 `720px` 到 `960px`，桌面端抽屉宽度为窗口宽度的 `56%` 且最小 `640px`。

## 服务端 Web 控制台

服务端采用左侧导航、顶部工具栏和内容区。主导航包括工作台、设备管理、采集任务、点位缓存、规则过滤、北向转发、插件中心、日志诊断和系统设置。底部导航包括 APIKey、用户权限和关于系统。

- 工作台展示运行设备、启用点位、运行任务、插件进程、转发速率、实时吞吐、系统健康、最近告警和插件状态。
- 设备管理使用设备表格和右侧抽屉，抽屉包含基本信息、设备配置、点位表、运行状态和调试日志。设备配置页不编辑点位表。
- 点位缓存只展示最新点位缓存，不提供历史查询；当前值列使用等宽数字。
- 插件中心突出本地插件导入、启停、升级、回滚和调试，在线应用商城只作为弱入口。
- 日志诊断支持来源、级别、插件、设备、任务和时间范围筛选，日志流支持暂停、继续、导出和清空视图。

## 桌面端页面

桌面端面向本机部署、边缘采集、现场调试和轻量运维，基础视口为 `1280x800`。默认框架为左侧窄导航、顶部本机状态栏、内容区和底部状态条。

- 顶部状态栏展示本地运行时、数据库、本地 HTTP API、插件进程和网络状态。
- 设备页采用左侧设备列表和右侧设备详情加底部点位表，是桌面端首要体验模板。
- 设备配置默认显示常用字段，高级字段放入高级设置；主操作只保留保存和测试连接。
- 点位缓存页用于现场实时观察，值变化后使用轻量高亮，持续不超过 `1.2s`。
- 日志页默认展示本机日志，支持暂停、清屏、导出和打开日志目录。
- 设置页包含本地运行时、数据库、本地 HTTP API、APIKey、插件目录、主题和关于。

## 状态与权限

- 操作成功使用右上角轻提示。
- 操作失败展示错误提示和可展开详情。
- 长任务展示进度条或步骤。
- 插件启动展示启动中状态和超时提示。
- 最新点位变化使用轻量高亮。
- 日志流默认自动滚动并支持暂停。
- 实时图表默认 5 秒刷新。
- 连接测试结果在设备页就地显示，不跳转到日志页。
- 只读账号隐藏新增、编辑、删除、启停、导入、回滚按钮，表格操作列只保留查看，表单进入只读态。
- 可编辑账号可访问配置和管理操作，危险操作仍需要确认。
- APIKey 页面状态包含正常、即将过期、已过期、已吊销和最近异常访问。

## 插件自定义页面

插件自定义页面嵌入时，宿主外层必须保留标题、说明、保存状态和审计提示。容器顶部显示插件名称、版本、权限和联网提示。黑夜模式通过宿主消息通道同步给插件页面。

以下异常状态由宿主统一展示：

- 插件页面加载失败。
- 插件页面版本不兼容。
- 插件页面请求越权。
- 插件页面保存失败。

## 可用性检查

- 浅色和黑夜模式文字对比度足够。
- `1280px` 宽度下表格列不严重挤压。
- 按钮文字不换行溢出。
- 抽屉底部操作始终可见。
- 日志和表格长文本有省略和详情查看。
- 状态颜色必须配合文字或标签。
