# 前端模块页面模式

## 常见文件

- `src/api/` 或现有 API 目录：TypeScript 请求/响应类型和 API 方法。
- `src/views/`：页面实现。
- `src/router/`：路由入口和标题元信息。
- `src/stores/`：跨页面共享状态。
- `src/i18n/` 或现有国际化入口：用户可见文本。

## 页面形态

标准管理页通常包含：

- 筛选区：紧凑表单、搜索、重置和常用筛选。
- 数据区：`n-data-table`、批量操作、分页和刷新。
- 编辑区：抽屉或弹窗，包含 `n-form`、校验规则、保存状态和成功反馈。
- 状态区：空状态、错误状态、加载状态和只读权限提示。

## Script 组织

推荐顺序：

1. `useRouter`、`useMessage`、`useDialog`、store、权限和 i18n。
2. 响应式状态：加载、保存、筛选、列表、分页、弹窗、表单。
3. 计算属性：列定义、选项、标题、校验规则。
4. 方法：加载、搜索、重置、分页、打开编辑、提交、删除。
5. 生命周期。

## API 处理

- 响应为空时提供稳定默认值。
- catch 中只记录带模块前缀的日志，并做本地状态回退。
- 不重复弹出统一拦截器已处理的错误。
- 成功提示和确认弹窗由业务代码控制。

## TypeScript

- 简单 ref 使用推断。
- 表单、API 参数、复杂行数据使用明确 interface。
- 避免 `any`，已知状态使用联合类型。
